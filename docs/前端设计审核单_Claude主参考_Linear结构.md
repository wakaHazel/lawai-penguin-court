# 前端设计审核单：Claude 主参考 + Linear 结构

更新时间：2026-04-09

## 1. 本次结论

本项目的前端主风格，确定为：

- 视觉语言：`Claude`
- 页面结构：`Linear`
- 最终落地：`法律卷宗感 + 产品工作台感`

不是做成：
- 通用 AI 聊天网站
- 花哨科技感官网
- 泛 SaaS 仪表盘

而是做成：
- 温暖克制的法律 AI 工作台
- 有流程主轴的专业系统
- 可演示、可扩展、可接元器工作流的正式产品骨架

## 3. 视觉上要保留什么

### 3.1 保留 Claude 的部分

- 暖纸张底色，而不是纯白后台
- serif 标题的权威感
- 暖灰正文，而不是冷灰科技感
- terracotta 作为唯一主强调色
- 整体克制、安静、可信
- 像编辑部和高质量文档，而不是像 AI 炫技页

### 3.2 保留 Linear 的部分

- 工作台式信息结构
- 明确的顶部上下文
- 清晰的阶段进度
- 主工作区 + 侧边上下文区
- 操作优先级非常明确
- 页面像“在处理任务”，不是“在看宣传”

## 4. 明确不做什么

- 不做紫色 AI 风
- 不做大面积发光渐变
- 不做三列平均卡片首页
- 不做一堆悬浮数据卡
- 不做普通企业后台 admin 模板味
- 不做赛博战情室风格
- 不做过度滚动动画

## 5. 页面结构判断

后续页面建议这样理解：

### 首页 / 入口区

不是营销首页。

它应该更像“项目总入口 + 当前案件启动台”：
- 有系统名
- 有一句非常克制的系统定位
- 有案件入口
- 有主轴流程可视化
- 有当前案件/当前状态的承接

### 主工作台

主工作台是整个系统核心。

建议结构：
- 顶部：项目名、案件标题、当前阶段、ID 信息
- 进度带：五段主轴
- 左侧：当前阶段主面板
- 右侧：案件摘要、争点、证据、缺口、提示
- 底部或局部：推进动作

## 6. 各阶段的页面气质

### 案件录入

像在建立卷宗，不像在填普通后台表单。

### 庭审模拟

像法庭当前场景，必须是页面视觉中心。

### 对方行为模拟

像对方策略预判简报，不要做成军事终端。

### 胜诉率分析

像审理研判结果，不要做成金融看板。

### 复盘报告

像正式可导出的法律工作成果。
