# 腾讯工具链与官方工具白名单检查清单

> 版本：2026-04-06

## 1. 官方工具白名单

比赛对外材料中，只能把以下 6 项写成核心开发工具：

- `腾讯开悟比赛平台`
- `CodeBuddy`
- `腾讯元器`
- `腾讯元宝`
- `小理AI`
- `得理开放平台API`

检查项：

- [ ] 项目简介中已明确写出 6 项官方工具
- [ ] 技术方案中已明确写出 6 项官方工具
- [ ] 答辩 PPT 中已明确写出 6 项官方工具
- [ ] 演示视频脚本中已明确写出 6 项官方工具
- [ ] AI工具使用说明中已明确写出 6 项官方工具

## 2. 腾讯开悟比赛平台

检查项：

- [ ] 已注册并可正常登录
- [ ] 已阅读赛题说明
- [ ] 已确认材料提交流程
- [ ] 已确认 demo 链接/作品链接提交方式
- [ ] 已确认视频与附件上传要求

## 3. CodeBuddy

检查项：

- [ ] 已作为核心开发协同工具纳入说明材料
- [ ] 已明确它在项目中的职责：代码开发、工程协同、联调支持
- [ ] 项目方案中未弱化或遗漏 CodeBuddy

## 4. 腾讯元器

检查项：

- [ ] 已明确元器是工作流编排主载体
- [ ] 已规划至少 1 套可展示的工作流
- [ ] 已准备工作流截图或流程图
- [ ] 已准备工作流节点说明
- [ ] 已准备工作流中使用的 Prompt 说明

## 5. 腾讯元宝

检查项：

- [ ] 已明确元宝在项目中的作用
- [ ] 已用于 Prompt 优化、文案生成、内容润色或材料辅助
- [ ] 已准备可写入材料的使用说明

## 6. 小理AI

检查项：

- [ ] 已明确小理AI是辅助法律实务参考工具
- [ ] 已准备至少 1 个参考使用场景
- [ ] 未将小理AI描述成项目唯一核心引擎

## 7. 得理开放平台API

检查项：

- [ ] 已确认案例检索接口
- [ ] 已确认法规检索接口
- [ ] 已确认 lawInfo 细节接口
- [ ] 已确认认证方式
- [ ] 已确认元器插件接入路径
- [ ] 已确认 `ArrayString` 与 `null` 风险规避方案

## 8. 对外表述禁区

以下说法应避免出现在比赛主材料中：

- [ ] 把第三方通用 agent 框架写成主工具链
- [ ] 把常规工程框架写成比赛核心创新工具
- [ ] 把项目描述成“泛法律聊天机器人”
- [ ] 把元器降级成普通模型调用接口

## 9. 对外标准表述

建议统一使用：

`本项目基于腾讯开悟比赛平台、CodeBuddy、腾讯元器、腾讯元宝、小理AI与得理开放平台API完成设计、开发、联调与展示。`
