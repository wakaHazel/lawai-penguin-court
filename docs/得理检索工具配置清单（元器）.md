# 得理检索工具配置清单（元器）

## 1. 使用原则

这 3 个工具需要在元器里单独创建：

1. `search_case`
2. `search_law`
3. `get_law_info`

统一原则：

- `appid` 和 `secret` 都放在 `Header`
- 不依赖平台统一 `ApiKey` 去赌双 Header 自动适配
- 配完后每个工具都要点一次 `去校验`

## 2. 工具一：search_case

### 基本信息

- 工具名：`search_case`
- 描述：`按关键词检索案例列表`
- Method：`POST`
- URL：`https://openapi.delilegal.com/api/qa/v3/search/queryListCase`

### Header

- `appid`
- `secret`

### Body

```json
{
  "pageNo": 1,
  "pageSize": 5,
  "sortField": "correlation",
  "sortOrder": "desc",
  "condition": {
    "keywordArr": ["{{keyword}}"]
  }
}
```

### 关键约束

- `keywordArr` 必须是数组，不是字符串。
- 如果前面节点产出的是单个字符串，必须先转成数组再喂给它。

### 最小测试

```text
工伤 交通事故 上下班途中
```

## 3. 工具二：search_law

### 基本信息

- 工具名：`search_law`
- 描述：`按关键词检索法律法规`
- Method：`POST`
- URL：`https://openapi.delilegal.com/api/qa/v3/search/queryListLaw`

### Header

- `appid`
- `secret`

### Body

```json
{
  "pageNo": 1,
  "pageSize": 5,
  "sortField": "correlation",
  "sortOrder": "desc",
  "condition": {
    "keywords": ["{{keyword}}"],
    "fieldName": "semantic"
  }
}
```

### 关键约束

- 这里用的是 `keywords`，不是 `keywordArr`。
- 返回的是法规列表，不是法规全文。

### 最小测试

```text
劳动合同 双倍工资 未签订书面合同
```

## 4. 工具三：get_law_info

### 基本信息

- 工具名：`get_law_info`
- 描述：`根据 law_id 查询法规详情`
- Method：`GET`
- URL：`https://openapi.delilegal.com/api/qa/v3/search/lawInfo?lawId={{law_id}}&merge=true`

### Header

- `appid`
- `secret`

### Query

- `lawId`：来自 `search_law` 结果里的法规 ID
- `merge`：固定 `true`

### 关键约束

- `search_law` 只给列表摘要，拿正文必须继续调 `get_law_info`。
- `law_id` 必须来自真实检索结果，不要手写假值联调主流程。

## 5. 配完后的核对清单

每个工具都检查这 5 项：

1. 请求方法对不对。
2. URL 对不对。
3. `appid` 和 `secret` 是否真的在 Header。
4. 参数名是否拼对。
5. `去校验` 是否通过。

## 6. 这 3 个工具和工作流的关系

这 3 个工具是 `工具层`，不是当前 workflow zip 里自动带出来的内容。

正确关系是：

- 先在元器里建好这 3 个工具
- 再让 `W02_法律支持检索` 和主智能体去调用它们
- 最后再把结果接回 `W00_企鹅法庭主控编排`

## 7. 当前项目建议接法

建议至少这样接：

1. `W02` 先生成 `law_keyword` 和 `case_keyword`
2. 插件节点调用 `search_law`
3. 插件节点调用 `search_case`
4. 如果法规命中质量高，再追加 `get_law_info`
5. 用代码节点或变量聚合把多路结果合并
6. 最后再给大模型做总结输出

## 8. 最后一句提醒

如果你发现：

- 工具能单测通过
- 但工作流里还是没有结果

优先检查的不是 API，而是：

1. 工作流里插件节点有没有真的选中对应工具
2. 输入变量有没有绑到正确字段
3. 多分支结果有没有先经过聚合再进回复/封装节点
