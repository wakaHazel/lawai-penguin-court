# 元器最小回传样例需求（变量面板）

## 1. 这份样例是为了解决什么

当前还没有被彻底打通的问题只有一个：

- 元器导入后，`工作流级变量 / 启动工作流输入变量` 面板到底由哪部分导出数据稳定驱动

现有结论是：

- 单看 workflow json 不够
- 单看 `参数.xlsx` 和 `变量.xlsx` 也不够
- 真实用户导出的包里，这几块信息有时并不会完整保留

所以现在最需要的是一个最小、干净、人工确认过的元器导出样例。

## 2. 请按这个最小规模去搭

不要导出大工作流，只搭一个最小样例：

1. 新建一个空工作流
2. 画布上只放：
   - `开始节点`
   - `代码节点`
   - `结束节点`
3. 不要加额外插件、子工作流、分支、回复节点

## 3. 必须手工配置的内容

### 开始节点

在 `API 参数 -> 管理` 里手工加 2 个参数：

1. `case_id`
2. `branch_focus`

建议都填默认值：

- `case_id = CASE-001`
- `branch_focus = 劳动关系是否成立`

### 工作流级变量

手工新增 2 个工作流级变量：

1. `querytips`
2. `debug_flag`

建议默认值：

- `querytips = 本工作流仅用于案例法规检索`
- `debug_flag = on`

### 代码节点

代码节点只做最简单回传：

```python
def main(params: dict) -> dict:
    return {
        "case_id": params.get("case_id", ""),
        "branch_focus": params.get("branch_focus", ""),
        "querytips": params.get("querytips", ""),
        "debug_flag": params.get("debug_flag", "")
    }
```

### 结束节点

结束节点把这 4 个字段都绑定出来：

1. `case_id`
2. `branch_focus`
3. `querytips`
4. `debug_flag`

## 4. 导出前要确认

在元器界面里，你要先肉眼确认：

1. `开始节点` 里真能看到这 2 个 API 参数
2. `工作流级变量` 面板里真能看到这 2 个变量
3. `结束节点` 输出里真绑定了 4 个字段

只有确认 UI 里已经显示正常，再导出。

## 5. 导出后给我什么

给我整个导出包即可，最好包含：

- workflow json
- `参数.xlsx`
- `变量.xlsx`
- `工作流程.xlsx`
- `工作流程引用.xlsx`
- `示例问法.xlsx`

## 6. 这个样例为什么重要

只要拿到这个最小样例，我就可以做三件事：

1. 对照当前生成器差异定位真正缺字段
2. 判断 `变量.xlsx` 到底是不是关键驱动
3. 再把正式包里的变量面板兼容性继续往前推进

## 7. 一句话要求

请不要导出“大而全”的复杂样例，只导出这个最小样例。

越小，越容易把变量面板问题一次性定位清楚。
