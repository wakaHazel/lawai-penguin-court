# 法律专业性与合规治理检查清单

> 版本：2026-04-06

## 1. 法律内容来源

- [ ] 法条依据来自现行有效法律法规
- [ ] 司法解释依据来自现行有效司法解释
- [ ] 类案依据来源可追溯
- [ ] 不使用来路不明的法律结论作为最终依据

## 2. 法律内容审校

- [ ] 至少 2 名法学同学参与审校
- [ ] 关键法条引用已人工复核
- [ ] 关键争点划分已人工复核
- [ ] 关键证据判断已人工复核
- [ ] demo 案件事实与推理逻辑已人工复核

## 3. 输出边界

- [ ] 系统输出明确为辅助分析，不包装成正式法律意见书
- [ ] 报告中区分“法条依据”“案例参考”“模拟建议”
- [ ] 对不确定结论给出风险提示
- [ ] 对缺失证据给出补充提示

## 4. AI 使用边界

- [ ] 小理AI定位为辅助参考，不作为唯一权威来源
- [ ] 得理开放平台API定位为法律检索原子能力
- [ ] 元器工作流输出经过结构化约束
- [ ] 高风险文本有模板或规则兜底

## 5. 数据安全与伦理

- [ ] 演示案例优先使用虚构或脱敏案例
- [ ] 不上传真实敏感个人信息到公开材料
- [ ] 作品演示中不暴露 API 凭证
- [ ] 文档中不硬编码 app secret

## 6. 比赛答辩中的禁区

- [ ] 不夸大为“替代律师”
- [ ] 不宣称“保证胜诉”
- [ ] 不宣称“完全自动给出法律结论”
- [ ] 不把 AI 输出表述成最终司法判断

## 7. 推荐答辩表述

建议统一表述为：

`本项目定位为面向法律场景的智能辅助系统，重点提升庭前准备、庭审模拟、案例检索与复盘分析效率，不替代专业律师、法官或正式法律意见。`
