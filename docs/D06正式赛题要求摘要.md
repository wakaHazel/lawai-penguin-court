# D06 正式赛题要求摘要

> 版本：2026-04-06  
> 适用项目：`企鹅法庭·沉浸式庭审模拟与立法沙盘推演AI智能体系统`

## 1. 赛题定位

- 赛题名称：`【D06】法律AI应用创新与实践`
- 赛题方向：`智慧法律应用类`
- 赛题目标：依托腾讯系 AI 工具平台，开发一个可在法律相关场景应用的智能体、应用或系统。

## 2. 可接受的作品类型

赛题明确允许三类作品：

- `智能体开发`
- `工具应用开发`
- `系统解决方案`

对本项目最合适的归类是：

- `系统解决方案`

项目对外表述建议固定为：

`基于腾讯系AI工具平台构建的沉浸式庭审模拟与法律辅助分析系统`

## 3. 官方工具白名单

赛题原文明确要求：

`限使用下述开发工具完成本赛题开发任务`

允许使用的工具只有以下 6 项：

1. `腾讯开悟比赛平台`
2. `CodeBuddy`
3. `腾讯元器`
4. `腾讯元宝`
5. `小理AI`
6. `得理提供的原子能力“开放平台API”`

## 4. 工具使用口径

后续所有比赛材料必须统一以下口径：

- `腾讯开悟比赛平台`
  - 作为赛事平台、资料学习、链路打通、提交与展示平台
- `CodeBuddy`
  - 作为核心代码开发与工程协同工具
- `腾讯元器`
  - 作为智能体工作流编排主载体
- `腾讯元宝`
  - 作为 Prompt 调优、文案生成、内容润色辅助工具
- `小理AI`
  - 作为法律实务案例参考与辅助分析工具
- `得理开放平台API`
  - 作为案例检索、法规检索等法律原子能力来源

## 5. 关键提交要求

赛题特别强调：

- 必须体现 `腾讯系AI工具平台` 的使用。
- 必须体现 `AI工具使用说明（工作流与prompt）`。
- 必须体现 `法律需求分析 -> 技术方案设计 -> 核心功能开发 -> 专业验证测试 -> 迭代优化演示` 的完整链路。

## 8. 当前对本项目的直接影响

对企鹅法庭项目来说，这意味着：

- 对外主叙事必须是 `腾讯元器 + 腾讯元宝 + CodeBuddy + 腾讯开悟平台 + 小理AI + 得理开放平台API`
- 不能把项目包装成“随意拼接第三方工具的法律 AI 平台”
- `工作流图` 与 `Prompt 说明` 必须单列，不可省略
- `法律专业人员参与验证` 必须写进方案与测试环节
- `demo` 必须优先保证可演示、可解释、可复现
