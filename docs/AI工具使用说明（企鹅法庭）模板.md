## 3. 元器工作流设计

### 3.1 工作流目标

本项目的元器工作流主要服务以下 3 类能力：

- 庭审流程交互
- 对方行为模拟
- 胜诉率分析

### 3.2 推荐工作流主链路

建议写成：

1. 用户输入案件基础信息
2. 元器识别案件类型与核心争点
3. 工具节点调用得理开放平台API检索法规与案例
4. 大模型节点生成当前庭审阶段内容
5. 对方行为模拟节点生成抗辩或新情况
6. 聚合节点整理案件状态
7. 输出节点生成：
   - 当前庭审文本
   - 用户可选策略
   - 风险提示
   - 胜诉率分析摘要

### 3.3 元器节点说明模板

| 节点名称 | 节点类型 | 输入 | 输出 | 作用 |
|---|---|---|---|---|
| 案件识别节点 | 大模型节点 | 案件信息 | 案件类型、争点 | 识别案件结构 |
| 法规检索节点 | 工具节点 | 关键词 | 法规结果 | 调用得理 API |
| 案例检索节点 | 工具节点 | 关键词 | 类案结果 | 调用得理 API |
| 庭审生成节点 | 大模型节点 | 案件状态 | 场景文本 | 输出交互内容 |
| 对方模拟节点 | 大模型节点 | 当前阶段 | 抗辩行为 | 生成对方动作 |
| 聚合节点 | 变量聚合/代码节点 | 多节点结果 | 统一状态对象 | 避免多分支空值 |
| 输出节点 | 回复节点 | 聚合结果 | 页面展示内容 | 输出给前端 |

## 4. Prompt 设计说明

### 4.1 Prompt 设计原则

建议强调 4 点：

- 法律专业性优先
- 结构化输出优先
- 不确定时给出风险提示
- 不输出替代律师或法官的绝对结论

### 4.2 三类核心 Prompt

#### A. 庭审流程交互 Prompt

职责：

- 生成当前阶段的庭审文本
- 限制在既定案件事实和阶段内推进
- 输出可选行动分支

建议约束：

- 不得跳过法定阶段
- 不得虚构关键事实
- 输出必须结构化

#### B. 对方行为模拟 Prompt

职责：

- 模拟对方当事人、代理人或证人的行为
- 生成高合理性抗辩内容

建议约束：

- 只能在案件设定与证据范围内生成内容
- 不能脱离法律逻辑胡乱制造冲突

#### C. 胜诉率分析 Prompt

职责：

- 结合争点、证据、法条、类案输出分维度分析

建议约束：

- 分维度解释
- 明确风险点
- 明确证据缺口
- 不做绝对承诺

## 6. 小理AI使用说明

建议写法：

`小理AI在本项目中主要作为法律实务应用案例参考与辅助分析工具，用于辅助理解法律问答、AI检索、合同生成与审查、文本分析、文书生成等法律AI应用形态，从而帮助团队优化企鹅法庭的交互设计与输出表达。`

## 7. 风险控制说明

建议在本节明确：

- AI 输出仅作为辅助分析
- 关键法律内容经法学同学人工审校
- 对缺失证据和不确定结论提供提示
- 不输出保证胜诉等不当表述
