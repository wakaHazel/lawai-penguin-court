> 版本：2026-04-06

> 适用项目：`企鹅法庭·沉浸式庭审模拟与立法沙盘推演AI智能体系统`

## 1. 项目简介

### 1.1 项目基础信息

建议固定写法：

- 项目名称：`企鹅法庭·沉浸式庭审模拟与立法沙盘推演AI智能体系统`
- 项目类型：`系统解决方案`
- 赛题方向：`智慧法律应用类`
- 项目定位：
  - 面向法律学习、庭前准备、法律服务辅助场景
  - 基于腾讯系AI工具平台构建的沉浸式法律 AI 应用
- 核心创新：
  - 文游式沉浸交互
  - 结构化庭审流程控制
  - 法律检索与胜诉率分析结合

### 1.2 项目需求分析

建议从 4 段写：

1. 行业问题
   - 案件激增
   - 法律服务资源不均
   - 普通用户庭前准备能力弱
   - 法律学习缺少沉浸式训练工具
2. 目标用户
   - 法律专业学生
   - 普通法律服务需求用户
   - 基层法律服务场景
3. 典型痛点
   - 不知道案件争点
   - 不知道证据缺什么
   - 不清楚对方可能抗辩
   - 无法低成本进行庭审预演
4. 项目价值
   - 提升法律服务可及性
   - 提升庭前准备效率
   - 提供沉浸式法律学习体验

### 1.3 解决方案思路

建议固定为：

- 前端负责：案件录入、模拟互动、分析展示
- 元器负责：智能体工作流编排
- 得理开放平台API负责：案例/法规检索
- 小理AI负责：法律实务辅助参考
- 后端负责：状态管理、规则控制、报告生成

### 1.4 关键功能说明

建议写 4 个核心功能：

- 案件信息可视化录入
- 庭审模拟交互
- 对方行为模拟
- 胜诉率分析与复盘报告

### 1.5 项目使用场景

建议写 3 个场景：

- 法律学生模拟庭审训练
- 当事人庭前准备辅助
- 法律服务机构低成本预咨询辅助

### 1.6 AI工具使用说明（工作流与Prompt）

这一节必须单独成章，建议写成：

  - 小理AI

## 2. 项目 Demo

### 2.1 作品 Demo

最低要求：

- 提供可运行演示链路
- 如果有公网链接，提供公网链接
- 如果没有公网链接，保证本地演示稳定可复现

建议演示链路固定为：

`案件录入 -> 启动模拟 -> 庭审推进 -> 对方抗辩 -> 胜诉率分析 -> 复盘报告`

### 2.2 Demo 演示视频

建议视频时长控制：

- `3 - 5 分钟`

建议镜头顺序：

1. 项目名称与一句话定位
2. 用户进入案件录入页
3. 启动庭审模拟
4. 展示对方行为模拟
5. 展示得理 API 检索结果如何支撑推理
6. 展示胜诉率分析
7. 展示复盘报告
8. 结束页强调腾讯工具链与法律价值

## 4. 初赛最容易扣分的地方

- 没有把作品明确归类为 `系统解决方案`
- 没把 `AI工具使用说明（工作流与Prompt）` 单独成章
- 只写产品功能，没有写法律需求分析
- 只写“用了 AI”，没有写具体用了哪些官方工具
- 演示视频只展示界面，不展示智能工作流和输出价值
- 法律专业性只靠口头说明，没有人工验证痕迹
