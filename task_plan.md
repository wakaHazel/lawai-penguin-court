# Task Plan: 企鹅法庭庭审模拟深改

## Goal
在不偏离主轴的前提下，把“庭审模拟”从目前的流程演示版，逐步改造成可补充新事实、可进行真实攻防、可输出细致诉讼建议的沉浸式模拟模块。

## Phases
- [x] Phase 1: 盘点现状问题与用户新增需求
- [x] Phase 2: 提取庭审笔录中的可复用庭审表达与节奏
- [x] Phase 3: 输出分阶段深改方案与优先级
- [x] Phase 4: 优先修交互主链与页面收口
- [ ] Phase 5: 接入“用户补充输入层”与细化攻防引擎
- [ ] Phase 6: 联动胜诉率分析与复盘报告
- [ ] Phase 7: 用真实 demo 案件与法律素材做联调打磨

## Key Questions
1. 如何把“全由系统生成”改成“系统推进 + 用户补充”的混合模拟？
2. 如何把“宏观建议”改成“可直接拿来开口、质证、补证、反驳”的实战建议？
3. 如何让对方模拟真正输出细致具体的论点、证据、话术、法条和说理路径？
4. 如何在 UI 上只保留当前阶段、当前问题、当前动作，去掉干扰信息？
5. 如何在模拟结束后自然跳转到胜诉率分析和复盘，不让链路断掉？

## Decisions Made
- 深改仍严格服从主轴：`案件录入 -> 庭审模拟 -> 对方行为模拟 -> 胜诉率分析 -> 复盘报告`。
- 不先扩案由，不先加无关模块，先把“庭审模拟”这一段做深做实。
- 后续模拟文本不再只追求氛围，而要以真实庭审笔录中的节奏、问法、质证表达、辩论结构为依据。
- “建议”模块后续必须拆成可执行条目，不能只给空泛方向。
- 后续每轮改造优先保证可演示、可验证，避免大改后整体失控。

## Errors Encountered
- PowerShell profile 会报脚本策略错误，但不影响当前工作，忽略即可。
- 通过 Python 直接按中文路径读取 `docx` 时出现编码干扰，已改为显式 Unicode 文件名方式读取。

## Status
**Currently in Phase 5** - 已完成第 1 轮“交互主链收口”，下一步进入“用户补充输入层”与攻防细化。

## Progress Update
- 已移除模拟页中阶段提示、角色标签、CG 元信息等明显“系统面板感”内容。
- 已把模拟态的叙事区压缩为最近两段，避免一轮内堆叠四段重复叙事。
- 已在模拟结束态加入明确入口：`去对方推演`、`去胜诉率分析`、`去复盘报告`。
- 已验证从首页进入新建案件、使用示例模板创建案件、推进到 `复盘就绪`、再跳到 `胜诉率分析` 的链路可用。
- 已修正演示模式下的默认 API 探测行为，前端单跑预览时不再产生首页或建案时的失败请求噪音。
