# 元器测试页诊断

时间：2026-04-15

测试链接：

- `https://yuanqi.tencent.com/webim/#/chat/xwiFKI?appid=2043273425069278272&experience=true`

## 已确认事实

1. 该链接可以正常打开，页面标题为“腾讯元器”。
2. 页面右侧能正常展示发布信息：
   - 模型：`混元大模型TurboS最新版`
   - 工作流：`W00_企鹅法庭主控编排`、`W04_胜诉率分析与复盘`、`W03_对方行为模拟` 等
   - 发布渠道：微信 / 元器
3. 页面底部输入框不是普通 `textarea`，而是 `contenteditable` 富文本编辑器。
4. 输入区外层存在前端遮罩：`.not-login-mask`。
5. 在未登录状态下，点击发送不会进入工作流执行，而是直接拉起登录流程。

## 关键证据

### 1. 未登录遮罩存在

输入区祖先节点中包含：

```html
<div class="question-input">
  <div class="not-login-mask"></div>
  ...
</div>
```

这说明当前测试页在未登录桌面环境下，只能“看见”发布信息，不能直接裸跑对话。

### 2. 点击发送后的真实行为

在强行移除遮罩并点击发送后，页面文本出现：

```text
欢迎登录腾讯元器
```

说明发送动作没有落到工作流执行，而是触发登录拦截。

### 3. 网络请求证据

发送后抓到的关键请求：

- 跳转登录：
  - `GET https://yuanqi.tencent.com/login?s_url=...&new_login=1`
- 登录页相关接口全部 401：
  - `GET https://yuanqi.tencent.com/api/getuserinfo`
  - `GET https://yuanqi.tencent.com/api/account/config`
  - `GET https://yuanqi.tencent.com/api/agent/config`
  - `GET https://yuanqi.tencent.com/api/model/config`
  - `GET https://yuanqi.tencent.com/api/agent/knowledge/config`
  - `POST https://yuanqi.tencent.com/api/v2/cgi?cmd=ListAgreement`

结论：当前无登录态时，根本没有进入工作流消息提交链路，所以也不可能在这个环境下验证工作流输出结构。

## 当前结论

这个测试页链接本身没有坏。

真正的问题是：

- 它在未登录桌面浏览器里是“只读展示 + 登录拦截”状态；
- 不能用来直接验证工作流是否执行、参数是否透传、返回是否为结构化 JSON。

也就是说，当前这个 URL **不能替代真正的已登录元器调试环境**。

## 对后续排查的意义

这次排查排除了一个误区：

- 不是我这边没点到输入框；
- 不是页面没有发送按钮；
- 不是工作流已经执行但我没抓到返回；
- 而是发送前就被登录拦截了。

所以后续如果要继续用这个“测试页”排查工作流，必须满足至少一个条件：

1. 在已登录腾讯元器的浏览器里打开这个链接再测。
2. 由你们本机实际登录后截图或录屏，把发送后的真实请求/真实返回给我。
3. 直接提供元器工作台里“工作流调试”页面的截图，而不是公开体验页。

## 建议下一步

优先建议：

1. 让做工作流的同学在她本机已登录状态下打开这个链接。
2. 发送一条最小测试消息，比如“请回复 ok”。
3. 截图三处内容给我：
   - 发送后聊天区完整返回
   - 如果有“调试详情/步骤执行详情”，把每一步展开
   - 浏览器开发者工具 Network 中真正的工作流请求与响应

如果她拿不到 Network，也可以至少给我：

- 发送前页面截图
- 发送后页面截图
- 右侧发布信息区截图
- 元器工作台里对应工作流的发布配置截图
