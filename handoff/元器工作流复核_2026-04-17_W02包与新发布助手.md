## 6. 现在应该怎么改

## 6.1 第一优先级：给真实业务分支补 `ANSWER` 节点

这是现在最该立刻改的。

不然即使 `prepare` 跑通，用户也还是看不到内容。

最简单的做法有两个：

### 做法 A：在 `代码2` 后面统一加一个 `ANSWER`

例如：

1. `代码2 -> 回复统一 -> 结束`

`回复统一` 输入：

1. `content <- 代码2.Output.final_out`

这样：

1. 所有真实分支汇总后都能给用户返回文本
2. fallback 也可以统一经过这里

### 做法 B：每个真实分支各自加 `ANSWER`

例如：

1. `代码12 -> 回复_prepare`
2. `代码1 -> 回复_trial`
3. `代码11 -> 回复_report`

优点：

1. 每个分支更清晰
2. 更适合后面做不同风格输出

我更推荐这个做法。

## 6.2 第二优先级：补齐剩余 3 个缺失 schema

必须补：

1. `W00.大模型2`
2. `W00.大模型3`
3. `W04.大模型1`

至少都要有：

```text
Output
  Thought: STRING
  Content: STRING
```

## 6.3 第三优先级：单测 `W02` 和 `W03`

现在中深分支统一 `504`，不要再先测主控。

建议顺序：

1. 单测 `W02`
2. 单测 `W03`
3. 单测 `W00` 中间分支
4. 单测 `W04`
5. 最后再测 `report_ready`
